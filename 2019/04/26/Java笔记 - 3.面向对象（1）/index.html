类和对象

面向对象的程序设计有两个重要概念：类（class）和对象（object，也被后才能为实例，instance）。类是一种概念，对象是实体。

  • 修饰符：可以省略，也可以是public、protected、private、static、final、abstract，其中public、protected、private只能出现一个，abstract和final最多只能出现一个，可以与static组合起来修饰方法、

方法详解

方法由函数发展而来，但方法与函数有着显著不同。在结构化编程里，函数是一等公民，程序由函数组成。但是在面向对象编程语言里，类是一等公民，系统由类组成。因此在JAVA语言里，方法不能独立存在，方法必须属于类或者对象。

Java语言里方法的所属性主要体现在以下几个方面：

  • 方法不能独立定义
  • 从逻辑上来看，方法要么属于该类本身，要么属于该类的一个对象
  • 永远不能单独执行方法，执行方法必须使用类或者对象作为调用者

使用static修饰的方法属于这个类本身，可以使用类本身来调用，也可以使用对象来调用。没使用static修饰的方法只能使用对象来调用。

Java方法的参数传递方式只有一种：值传递。将实际参数值的副本传入方法内，而参数本身不会受到任何影响。Java把内存分为两种：堆内存和栈内存。堆内存用来存储对象和数组，栈随着方法的开始而开辟，随着方法的结束而删除。当调用方法时，如果实际参数是调用者自己的栈内存中的值，那么它不会被改变，但如果参数是堆内存中的对象或者数组，那么调用者和被调用者都可以改变堆内存中的值。

成员变量和局部变量

成员变量指的是在类里定义的变量，局部变量指的是在方法里定义的变量。使用成员变量的时候应注意变量的所属，是类还是实例。调用类的成员变量时，尽量使用类作为主调。在使用变量时，要注意变量使用范围，然后在正确的范围里定义变量。

隐藏和封装

良好的封装是为了：

  • 隐藏类的实现细节
  • 在方法里加入控制逻辑，限制对属性的不合理访问
  • 可进行数据检查，从而有利于保证信息完整性
  • 便于修改，提高代码可维护性

实现封装的途径是：

  • 将对象的属性和实现细节隐藏，不允许外部直接访问
  • 把方法暴露出来，让方法操作和访问这些属性

要实现隐藏属性，就要了解Java的访问控制级别：由小到大依次为private、default、protected、public。

关于访问控制符的使用：

  • 类里的绝大部分成员都应该用private修饰，只有一些static修饰的、类似于全局变量的成员变量才使用public。
  • 有些方法用来辅助该类的其他方法实现，被称为工具方法，工具方法也应该使用private修饰。
  • 如果这个类主要用作其他类的父类，该类的绝大部分方法仅希望被其子类重写，而不希望被外界直接调用，则用protected修饰。
  • 希望暴露出来被外界自由调用的方法应该使用public修饰。
